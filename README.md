

# 🌌 PixelEncoder v6.4 (Post-Quantum Edition)

**PixelEncoder** — это утилита для надежного шифрования файлов и текста внутри .enc или PNG-изображений с использованием современных постквантовых алгоритмов шифрования.

Программа использует асимметричную криптографию **ML-KEM-768 (Kyber)** для защиты ключей и **AES-256-GCM** для шифрования самих данных. Никто не сможет извлечь ваши данные из .enc или картинки без правильного приватного ключа.

## 📦 Портативность

Программа собрана в **один `.exe` файл** — никакой установки, никаких зависимостей. [Скачал](https://github.com/Pikatyu8/image-encoder/releases/download/v6.4/pixel_encoder_pq.exe), запустил, работает. Можно носить на флешке, отправить другу, положить в любую папку.

## 🚀 Быстрый старт (Интерактивный режим)

Самый простой способ использовать программу — просто **запустить `pixel_encoder_pq.exe` двойным кликом**.

Откроется удобное интерактивное меню с вкладками:

Поддерживается перетаскивание файлов (Drag & Drop) в окно консоли, а также любые пути (относительные `./file`, абсолютные `C:\`, переменные среды `%USERPROFILE%\...`).

### Основной сценарий работы:

#### Шаг 1: Создание ключей (KeyGen)
Вкладка `[3] KeyGen`. Программа создаст два файла:
* 🔓 `public.kyber` — **Публичный ключ**. Отдайте тому, кто будет шифровать для вас.
* 🔐 `private.kyber` — **Приватный ключ**. Храните в секрете, он нужен **только вам** для расшифровки.

#### Шаг 2: Настройка профилей (Profiles)
Вкладка `[4] Profiles`. Здесь можно:
* **Настроить свой профиль** — указать имя и пути к вашим ключам. При расшифровке программа автоматически предложит ваш приватный ключ.
* **Добавить контакты** — сохранить публичные ключи других людей. При шифровании достаточно выбрать получателя из списка, а не вводить путь каждый раз.

Профили хранятся в `~/.pixelencoder/profiles.json`.

#### Шаг 3: Шифрование (Encode)
Вкладка `[1] Encode`.
1. Выберите получателя из контактов или укажите путь к публичному ключу вручную.
2. Укажите, что спрятать: файл (до 100 МБ) или текст.
3. Укажите формат итогового файла, .enc или .png (для наглядной визуализации шифрового шума, **более подозрительно при реальном использовании**).
4. Укажите имя итогового файла.

#### Шаг 4: Расшифровка (Decode)
Вкладка `[2] Decode`.
1. Укажите путь к файлу с секретом.
2. Программа предложит использовать ключ из вашего профиля, либо можно указать путь вручную.
3. Выберите папку для сохранения.

## 💻 Использование через командную строку (CLI)

Программа поддерживает полноценный CLI-режим для автоматизации и скриптов. Все команды доступны без интерактивного меню.

### Справка

```cmd
pixel_encoder_pq.exe --help
pixel_encoder_pq.exe encode --help
```

```cmd
pixel_encoder_pq.exe --version
pixel_encoder_pq.exe -V
```

### Генерация ключей

```cmd
pixel_encoder_pq.exe keygen [ДИРЕКТОРИЯ]
```

| Аргумент | Описание | По умолчанию |
|---|---|---|
| `ДИРЕКТОРИЯ` | Папка для сохранения `public.kyber` и `private.kyber` | `.` (текущая) |

**Примеры:**
```cmd
pixel_encoder_pq.exe keygen
pixel_encoder_pq.exe keygen ./my_keys
pixel_encoder_pq.exe keygen C:\Users\User\keys
```

### Шифрование

```cmd
pixel_encoder_pq.exe encode ПУБЛИЧНЫЙ_КЛЮЧ [ОПЦИИ]
```

| Параметр | Описание | По умолчанию |
|---|---|---|
| `ПУБЛИЧНЫЙ_КЛЮЧ` | Путь к `.kyber` файлу получателя (обязательный) | — |
| `--file`, `-f` | Путь к файлу для шифрования | — |
| `--text`, `-t` | Текстовая строка для шифрования | — |
| `--output`, `-o` | Имя выходного файла | `encoded.enc` |
| `--format` | Формат вывода: `enc` (бинарный) или `png` (изображение) | `enc` |

> ⚠ Необходимо указать **либо** `--file`, **либо** `--text`.

**Примеры:**
```cmd
:: Зашифровать файл → .enc (компактный бинарный формат)
pixel_encoder_pq.exe encode public.kyber -f secret.zip -o package.enc

:: Зашифровать файл → .png (данные в пикселях изображения)
pixel_encoder_pq.exe encode public.kyber -f photo.jpg -o hidden.png --format png

:: Зашифровать текстовое сообщение
pixel_encoder_pq.exe encode keys\public.kyber -t "Встречаемся в полночь" -o msg.enc

:: Полный путь к ключу
pixel_encoder_pq.exe encode C:\keys\bob_public.kyber -f report.pdf -o report.enc
```

### Расшифровка

```cmd
pixel_encoder_pq.exe decode ВХОДНОЙ_ФАЙЛ ПРИВАТНЫЙ_КЛЮЧ [ОПЦИИ]
```

| Параметр | Описание | По умолчанию |
|---|---|---|
| `ВХОДНОЙ_ФАЙЛ` | Файл `.enc` или `.png` для расшифровки (обязательный) | — |
| `ПРИВАТНЫЙ_КЛЮЧ` | Путь к приватному ключу `.kyber` (обязательный) | — |
| `--out-dir`, `-d` | Директория для сохранения расшифрованного файла | `.` (текущая) |
| `--force` | Перезаписывать существующие файлы без вопросов | `False` |

> Формат входного файла определяется **автоматически** по magic-байтам: `PXEN` → `.enc`, `\x89PNG` → `.png`.

**Примеры:**
```cmd
:: Расшифровать .enc файл
pixel_encoder_pq.exe decode package.enc private.kyber

:: Расшифровать .png файл в указанную папку
pixel_encoder_pq.exe decode hidden.png private.kyber -d C:\output

:: С автоматической перезаписью (для скриптов)
pixel_encoder_pq.exe decode msg.enc keys\private.kyber -d ./out --force
```

### Интерактивный режим (явный вызов)

```cmd
pixel_encoder_pq.exe interactive
```

Запускает интерактивное меню. Эквивалентно запуску без аргументов (двойной клик).

## 🔨 Сборка из исходников

Для самостоятельной сборки `.exe` нужны Python 3.11+ и зависимости:

```cmd
pip install pyinstaller typer rich pillow cryptography liboqs-python
```

Положите `oqs.dll` в папку `bundled_libs/` рядом со скриптом, затем:

```cmd
pyinstaller --onefile ^
    --add-binary "bundled_libs/oqs.dll;oqs_native/" ^
    --hidden-import oqs ^
    --hidden-import oqs.oqs ^
    --collect-submodules rich ^
    --collect-data rich ^
    pixel_encoder_pq.py
```

Готовый файл появится в `dist/pixel_encoder_pq.exe`.

## 📌 Важные примечания

* **Ограничение размера:** Максимальный размер скрываемого файла — **100 МБ**.
* **Размер картинки:** Итоговый PNG пропорционален размеру исходного файла.
* **Безопасность ключей:** Потеря `private.kyber` = невозможность восстановления данных.
* **Целостность:** Нельзя сжимать PNG с потерями (JPEG, мессенджеры без «отправить как файл»).

## 🛠 Устранение неполадок

Если при запуске ошибка с `oqs.dll` / `liboqs` — убедитесь, что DLL лежит рядом с `.exe`, либо задайте переменную `LIBOQS_DLL_DIR`.
